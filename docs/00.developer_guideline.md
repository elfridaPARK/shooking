## 3. 개발 환경 설정 및 규칙

### 🔧 개발 환경 설정

개발을 시작하기 위해 아래 단계를 따라 주세요:

1. 저장소 클론

   ```bash
   git clone https://github.com/your-org/your-repo.git
   ```
2. 패키지 설치

   ```bash
   npm install
   ```
3. 환경 변수 설정
   `.env.example`을 참고하여 `.env` 파일 생성
4. 로컬 서버 실행

   ```bash
   npm start
   ```

### 📁 폴더 구조

```
/src
  /components
  /pages
  /services
  /hooks
/tests
/docs
```

### 🌱 브랜치 전략 (예시)

* `main`: 배포용
* `develop`: 테스트/통합용
* `feature/*`: 기능 개발
* `hotfix/*`: 긴급 수정 사항

### 💬 커밋 메시지 규칙 (Conventional Commits)

```
feat: 새로운 기능 추가
fix: 버그 수정
docs: 문서 수정
refactor: 리팩터링
style: 코드 스타일 변경
test: 테스트 코드 추가/수정
```

## 5. 문서 참고 및 업데이트 가이드

### 📚 문서를 참고하는 방법

이 저장소에는 여러 문서가 있으며, 다음 순서대로 읽는 것을 추천합니다:

1. `developer-guide.md` → 전체 구조 이해

4. `coding-convention.md` → 코드 작성 규칙 숙지

1. 문서 수정 시 PR 생성
2. 문서 변경 내역을 `docs/CHANGELOG.md`에 기록
3. 팀 리뷰 후 병합
4. 문서 버전 태그 업데이트 (필요 시)

> 문서가 최신 상태를 유지해야 새로운 개발자가 **빠르게 온보딩**할 수 있습니다.
