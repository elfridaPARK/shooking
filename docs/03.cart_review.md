# 🛒 장바구니 페이지 리뷰

## 📌 고객사 요구사항

이메일에 명시된 기본 요구사항에 더해,  
피그마(Figma) 시안에서 확인된 세부 UI/UX 요구사항을 함께 반영하였습니다.

### ✅ 주요 요구사항 정리
1. **금액 확인 기능 강화**
   - 상품별 가격, 수량, 총합을 명확히 표시  
   - 결제 전 최종 결제 금액을 한눈에 확인할 수 있어야 함  
   - 배송비(₩3,000)는 기본 적용, **10만원 이상 구매 시 자동 무료 처리**

2. **사용자 편의성 개선**
   - 상품 수량을 조정할 때 (+, - 버튼) 실시간으로 금액이 변경되어야 함  
   - 장바구니 내 상품 삭제 기능 제공  
   - 뒤로가기 버튼 클릭 시 결제 페이지가 아닌 **상품 목록 페이지로 이동**  
   - 결제 버튼은 페이지 하단에 고정되어 항상 접근 가능하도록 유지  

3. **디자인 및 UI 정책**
   - 피그마 시안과 동일한 컴포넌트 배치 및 폰트 크기 유지  
   - 상품 리스트와 금액 요약 구역의 간격 및 여백 동일하게 구현  
   - “₩100,000 이상 구매 시 무료배송” 문구를 시각적으로 강조 표시  
   - 모바일(375px 기준) 중심 반응형 레이아웃 적용  
   - 버튼 및 주요 텍스트 색상은 브랜드 메인 컬러(#1E1E1E 계열) 사용  

4. **기술적 요구사항**
   - React 기반 SPA 환경에서 구성  
   - Axios를 통한 상품 정보 및 가격 데이터 호출  
   - 금액 계산 로직은 별도 커스텀 훅(`useCartCalculator`)으로 분리  
   - Context API로 장바구니 상태 전역 관리

## ⚙️ 요구사항에 따른 기능 동작 확인

### 🧩 1. 상품 리스트
- 각 상품은 이미지, 브랜드명, 단가, 수량 조절 버튼으로 구성됨  
- 수량 변경 시 즉시 금액이 업데이트되며, 총합 금액에도 반영됨  
- 상품 삭제 시 리스트에서 즉시 제거되고 금액이 자동 재계산됨  

### 💰 2. 금액 및 배송비 계산
- `총 상품 금액 = Σ(단가 × 수량)`  
- `총 결제 금액 = 총 상품 금액 + 배송비`  
- 배송비는 기본 3,000원이나, 총 상품 금액이 100,000원 이상일 경우 자동으로 0원 처리됨  
- 계산 로직은 `useCartCalculator` 훅에서 관리하며 상태 변화 시 자동 갱신됨  

### 🧾 3. 결제 프로세스
- “결제하기” 버튼 클릭 시 선택된 상품 정보(상품명, 수량, 금액)를 결제 모듈로 전달  
- 결제 API 호출 후 성공 시 결과 모달 표시, 실패 시 오류 메시지 출력  
- 결제 진행 중 중복 클릭 방지를 위한 로딩 인디케이터 표시  

### 🧭 4. 페이지 이동 및 내비게이션
- 상단 뒤로가기 버튼 클릭 시 상품 목록 페이지(`/product-list`)로 이동  
- 결제 성공 시 결제 완료 페이지(`/checkout/success`)로 리다이렉트  
- SPA 라우팅(React Router) 환경에서 페이지 새로고침 없이 전환됨  

### 📱 5. 반응형 디자인 동작
- 모바일 환경(375px 기준)에서 컴포넌트 간 여백, 버튼 크기 자동 조정  
- 하단 결제 버튼은 고정되어 스크롤 시에도 항상 화면에 표시됨  
- 데스크톱 환경에서는 좌우 여백을 넓혀 상품 정보 가독성 향상

## 🧠 개선 제안
- 결제 전 사용자 확인을 위한 “구매 요약 팝업” 추가 제안  
- 네트워크 오류 발생 시 재시도 버튼 제공  
- 장바구니 내 다중 선택 삭제 기능 추가 필요  
- 향후 포인트 적립 및 쿠폰 코드 입력 기능 확장 고려

## 🧩 요약 평가
본 장바구니 페이지는 고객 요구사항 및 피그마 시안을 충실히 반영하였으며,  
상품 정보 확인부터 결제 단계 진입까지 **명확하고 직관적인 사용자 경험(UX)** 을 제공함.  
기술적 구조 또한 재사용성과 확장성을 고려하여 설계되어,  
추후 기능 확장(예: 포인트/쿠폰/적립금)에도 유연하게 대응 가능함.
